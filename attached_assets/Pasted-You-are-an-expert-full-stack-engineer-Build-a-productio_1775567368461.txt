# 🧠 PROJECT OVERVIEW

Build a **full-stack web application** that allows users to:

* Organize their home into **hierarchical storage spaces** (rooms → cupboards → drawers → sections → etc.)
* Store and manage **items inside those storage spaces**
* Search and locate items quickly
* Scale this system for use in **homes, hotels, Airbnbs, and warehouses**

The system must be designed with **clean architecture, scalability, and extensibility in mind**.

# 🧩 CORE FEATURES

## 1. STORAGE HIERARCHY (CRITICAL)

Users can create nested storage spaces with **infinite depth**.

Example:
Household → Bedroom → Cupboard → Drawer → Box

Each storage must have:

* id
* name
* householdId
* parentId
* pathIds (list of ancestor IDs)
* pathNames (list of ancestor names)
* createdBy
* isPublic
* deleted (soft delete)

👉 Path arrays must be maintained automatically when creating storage.

---

## 2. ITEM MANAGEMENT

Users can add items into any storage.

Each item must have:

* id
* householdId
* storageId
* name
* category
* description
* tags (list)
* customFields (Map<String, Object>) ← dynamic metadata
* locationPathIds
* locationPathNames
* createdBy
* isPublic
* createdAt
* updatedAt
* deleted

👉 When item is created:

* Automatically inherit storage path
* Store full location path

## 8. API DESIGN

### Storage APIs

* POST /api/v1/storages
* GET /api/v1/storages/household/{householdId}
* GET /api/v1/storages/{parentId}/children
* DELETE /api/v1/storages/{id}

### Item APIs

* POST /api/v1/items
* GET /api/v1/items/household/{householdId}
* GET /api/v1/items/storage/{storageId}
* DELETE /api/v1/items/{id}
